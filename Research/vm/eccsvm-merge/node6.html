A Greedy Cluster Merging Algorithm

 

In this section, we use the error bounds derived in this paper to design a merging algorithm to control the exponential growth of the problem size. The name of the algorithm, ``greedy cluster merging,'' is fairly descriptive. One begins by clustering computational elements together into small subgroups based on proximity. Such clustering can be performed in O(N) computations in many practical instances (see [6] for example). The algorithm then walks through each cluster seeking to find opportunities to add elements to a mergeable collection. An element is added only if the merging operation induces an acceptable error. The element always takes adds an element to the collection when it can, hence the algorithm is greedy.
